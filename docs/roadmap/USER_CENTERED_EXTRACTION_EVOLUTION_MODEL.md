# User-Centered Extraction Evolution Model

Wave 0 status: model initialized from the enhancement prompt.

The product should evolve in controlled stages:

1. Make the current extraction evidence observable and reproducible.
2. Establish a typed geometry contract across extraction, rendering, overlays, and persistence.
3. Reconstruct words and lines from source-character evidence rather than text-run assumptions.
4. Add user correction workflows only after the object model can represent source evidence and derived objects separately.
5. Convert accepted corrections into declarative, reversible extraction knowledge.
6. Add automation only when geometry, confidence, provenance, and validation gates pass.

Current user-visible limitation: the app displays semantic word overlays, but those boxes are estimated from text fragments and font size. The UI does not yet explain this uncertainty or expose source evidence by default.
